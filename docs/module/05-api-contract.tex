\subsection{API Contract}\label{sec:api}
All endpoints follow RESTful conventions with JSON request/response bodies and standard HTTP status codes.
Each exchange shows a raw HTTP request or response trace: status line, headers, a blank line, then the body.
All paths are relative to the API base URL (\texttt{http://localhost:8080}).


\noindent\textbf{Authentication.} Protected endpoints require an \texttt{Authorization: Bearer <token>}
header. The token is a signed JWT issued by \texttt{POST /api/auth/login}. Requests without a valid
token receive \texttt{HTTP 401}.


\noindent\textbf{Ownership.} Users can only access their own resources. Attempting to read, update,
or delete another user's task or subtask returns \texttt{HTTP 403 Forbidden}.


\noindent\textbf{DELETE responses.} A successful delete returns \texttt{HTTP/1.1 204 No Content} with no
response body---the server has fulfilled the request and there is no representation to return (RFC~9110).
Error cases (e.g.\ resource not found) return \texttt{4xx} with a JSON body as documented below.

\subsubsection{Account Creation}
Use the resource-creation flow template for this \texttt{POST} endpoint (Figure~\ref{fig:seq-register}).

\textbf{\texttt{POST /api/auth/register}}

\begin{httpexchange}
\begin{httplisting}
POST /api/auth/register HTTP/1.1
Content-Type: application/json

{
    "username": "String (5-18 chars)",
    "password": "String (8+ chars, uppercase, lowercase, digit, special)"
}
\end{httplisting}
\httpmidarrows
\begin{httplisting}
HTTP/1.1 201 Created
(empty body)

HTTP/1.1 400 Bad Request
Content-Type: text/plain

Username must not be blank.
Username must be between 5 and 18 characters.
Password must not be blank.
Password must contain at least one special character (!@#$%^&*).
Username 'example' is already taken.
\end{httplisting}
\end{httpexchange}

\subsubsection{Authentication}
Use the login/auth flow template for this \texttt{/auth} endpoint (Figure~\ref{fig:seq-login}).

\textbf{\texttt{POST /api/auth/login}}

\begin{httpexchange}
\begin{httplisting}
POST /api/auth/login HTTP/1.1
Content-Type: application/json

{
    "username": "String",
    "password": "String"
}
\end{httplisting}
\httpmidarrows
\begin{httplisting}
HTTP/1.1 200 OK
Authorization: Bearer <signed-jwt-token>
Content-Length: 0
(empty body)

HTTP/1.1 401 Unauthorized
Content-Type: text/plain

Invalid username or password
\end{httplisting}
\end{httpexchange}


\noindent\textbf{JWT Claims:} The issued token contains \texttt{sub} (username),
\texttt{userId} (UUID string), \texttt{iat} (issued-at epoch seconds), and
\texttt{exp} (expiration = iat + 24 hours).

\subsubsection{Common Error Responses}

\begin{tabularx}{\textwidth}{@{} l l X @{}}
  \toprule
  \textbf{Status} & \textbf{Content-Type} & \textbf{Meaning} \\
  \midrule
  \texttt{400} & \texttt{application/json} or \texttt{text/plain} & Validation failure (blank title, weak password, etc.) \\
  \texttt{401} & \texttt{text/plain} & Missing, malformed, or expired token \\
  \texttt{403} & \texttt{application/json} & Authenticated user does not own the requested resource \\
  \texttt{404} & \texttt{application/json} & Resource (task or subtask) does not exist \\
  \texttt{503} & \texttt{text/plain} & Database temporarily unavailable \\
  \bottomrule
\end{tabularx}


\noindent\textbf{JSON error body format} (for 400, 403, 404 from protected endpoints):
\begin{lstlisting}[style=json]
{
    "status": <http-status-code>,
    "message": "<human-readable description>"
}
\end{lstlisting}

\noindent\textbf{Plain text error body} (for 401, registration 400, 409, 503):
the response body is a single plain-text string describing the error.
