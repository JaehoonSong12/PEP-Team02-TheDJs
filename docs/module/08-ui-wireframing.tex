\subsection{UI Wireframing}\label{sec:wireframes}
Low-fidelity wireframes define the primary screens and navigation paths for the Angular frontend.
Each view maps to the user stories in Section~\ref{sec:overview}; arrows in the interaction overview
show how the client moves between routes after register, login, logout, and dashboard actions.

\subsubsection{Screen Navigation}
Figure~\ref{fig:wireframe-navigation} summarizes the application flow. Unauthenticated users land on
the home (login) screen; \textbf{SIGN UP} routes to register and \textbf{LOG IN} or successful
\textbf{CREATE ACCOUNT} routes to the dashboard. \textbf{LOG OUT} returns the user to login.

\begin{figure}[htbp]
  \centering 
  \includegraphics[width=\linewidth]{Wireframes/Interactions.png}
  \caption{Screen navigation --- home, register, and dashboard}
  \label{fig:wireframe-navigation}
\end{figure}

\subsubsection{Task Dashboard}
Figure~\ref{fig:wireframe-dashboard} covers Epic~3 (task management) and Epic~4 (subtask organization).
The dashboard lists primary todos with expandable groups, completion progress (e.g.\ \texttt{1/3}),
checkboxes, and delete controls. Users add top-level tasks via \textbf{+ ADD TASK} and nested items
via \textbf{+ ADD SUBTASK}. The header shows the authenticated username and a \textbf{LOG OUT} action.
