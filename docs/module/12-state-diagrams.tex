\subsection{Login and Session State}\label{sec:session-state}
Epic~2 specifies session-based authentication (AUTH-09): protected routes require an active session,
and unauthenticated access returns HTTP~401 (AUTH-10). Figure~\ref{fig:state-session} models the
client session lifecycle---the states the Angular application maintains between
\texttt{POST /api/auth/login}, successful todo/subtask calls, logout, and session expiry.

\begin{figure}[htbp]
  \centering
  \resizebox{0.92\linewidth}{!}{%
  \begin{tikzpicture}
    \begin{umlstate}[name=session]{Client Session}
      \umlstateinitial[name=start]
      \umlbasicstate[name=guest, y=-3.5, width=15ex]{Unauthenticated}
      \umlbasicstate[name=auth, x=9, y=-3.5, width=15ex,
        entry=create session, do=access todos/subtasks, exit=clear session]{Authenticated}

      \umltrans{start}{guest}
      \umlHVtrans[arg=login OK]{guest}{auth}
      \umlHVtrans[arg=logout / 401]{auth}{guest}
      \umltrans[recursive=-25|-55|2.2cm, recursive direction=right to bottom, arg=register OK, pos=1.3]{guest}{guest}
      \umltrans[recursive=155|125|2.2cm, recursive direction=left to top, arg=login fail, pos=1.3]{guest}{guest}
      \umltrans[recursive=25|55|2cm, recursive direction=right to top, arg=API OK, pos=1.2]{auth}{auth}
    \end{umlstate}
  \end{tikzpicture}%
  }
  \caption{Login/session state --- unauthenticated until login succeeds; \texttt{401} or logout returns to guest}
  \label{fig:state-session}
\end{figure}

Registration (\texttt{POST /api/auth/register}, HTTP~200) succeeds while the client remains
\emph{unauthenticated} until a separate login establishes the session. Login failure returns
HTTP~401 and leaves the client in \textbf{Unauthenticated}. Protected todo and subtask endpoints
keep the client in \textbf{Authenticated}; any \texttt{401} from the API or an explicit logout
transitions back to \textbf{Unauthenticated}, matching Figure~\ref{fig:seq-login} and the use case
\emph{include} on Log In (Section~\ref{sec:use-cases}).
